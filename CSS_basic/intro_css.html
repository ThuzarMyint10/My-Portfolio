Pseudo-classes ဟာဆိုရင် classes တွေနဲ့ similar ဖြစ်ပါတယ်။ မတူတာကတော့ သူဟာ element တစ်ခုရဲ့ class attribute ကို explicitly define တော့ မလုပ်ပါဘူး။ ဒါပေမဲ့ selectors တွေအားလုံးဟာ user interaction ပေါ်မှာပဲ အခြခံကြတာပဲဖြစ်ပါတယ်။ Pseudo-class ဟာဆိုရင် keyword တစ်ခုဖြစ်ပြီးတော့ element တစ်ခုရဲ့ style ကို selector တစ်ခုအနေနဲ့ အသုံးပြုနိုင်ပါတယ်။ Mozilla Developer Network or MDN docs မှာလေ့လာနိုင်ပါတယ်။ အဲ့မှာဆိုရင် pseudo-classes တွေအများကြီးရှိပါတယ်။ အဲ့ထဲကမှ ကျွန်တော် နဲနဲကို အသုံးပြုပုံနဲ့အတူ ဘယ်လို effect ဖြစ်တယ်ဆိုတာကို ပြောပြပေးသွားမှာဖြစ်ပါတယ်။ Link history လို့ခေါ်တဲ့ pseudo-classes ဟာဆိုရင်တော့ click or visited လုပ်လိုက်တဲ့အချိန်မှာ user ဖြစ်ချင်တဲ့ style ကို create လုပ်နိုင်ပါတယ်။ အဲ့ဒါဟာဆိုရင် ကျွန်တော်တို့ရဲ့ page ထဲမှာရှိတဲ့ Links တွေကို style ချတာပဲဖြစ်ပါတယ်။ သူ့ကိုရေးတဲ့အခါမှာ fullcolon ခံပြီးတော့ အဲ့နောက်မှာ pseudo-class ကိုရေးပေးရမှာဖြစ်ပါတယ်။ ဥပမာ("a:link{}") ပြီးရေးရမှာဖြစ်ပါတယ်။ နောက်ထပ် pseudo-class တစ်ခုကတော့ visited ပဲဖြစ်ပါတယ်။ သူဟာ လည်း visited or link လုပ်လိုက်တဲ့အချိန်မှာ style ထည့်ချင်တဲ့အခါမှာအသုံးပြုနိုင်ပါတယ်။ သူ့ကိုလည်း link လိုမျိုး create လုပ်ပေးရမှာဖြစ်ပါတယ်။ သူ့ထဲမှာဆိုရင်လည်း user တွေကြိုက်တဲ့ css property ကိုအသုံးပြုနိုင်ပါတယ်။ နောက်တစ်ခုကတော့ active pseudo-classes ပဲဖြစ်ပါတယ်။ သူဟာလည်း user interaction ပေါ်မှာအခြေခံပြီးတော့ apply လုပ်ပါတယ်။ နောက်ထပ်ကတော့ hover pseudo-classes ပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ element ပေါ်မှာ hover လုပ်လိုက်တဲ့အချိန်မှာ color တွေ change သွားတဲ့ property တွေကို လုပ်ဆောင်နိုင်ပါတယ်။ ကျန်တော်အခုပြောပြခဲ့တဲ့ pseudo-classes တွေကို html ရဲ့ apply လုပ်ချင်တဲ့ element မှာ apply လုပ်နိုင်ပါတယ်။

Understanding Values and Units

CSS property တိုင်းမှာဆိုရင် သူတု့ိကို accept လုပ်တဲ့ values တွေရှိပါတယ်။ ကျွန်တော်တို့ဟာ key words တစ်ခုကို သုံးပြုမယ်ဆိုရင် value ကိုလည်း describe လုပ်ပေးရမှာဖြစ်ပါတယ်။ ဥပမာ color ဆိုရင် blue ဆိုပြီးတော့ စသည်အားဖြင့် value ကိုပါထည့်ပေးရမှာဖြစ်ပါတယ်။ Value တစ်ခုကိုဖော်ပြတဲ့အခါမှာ ကျွန်တော်တို့ဟာ number, length, image, or URL တို့နဲ့လည်း ဖော်ပြနိုင်ပါတယ်။ အဲ့ values တွေဟာဆိုရင် categorized လုပ်ထားတဲ့ data types တွေပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ good reference တစ်ခုလိုပါလိမ့်မယ်။ Good reference ဆိုတာ common values တွေကို cover လုပ်ပြီးတော့ units CSS properties တွေကို accept လုပ်တာပဲဖြစ်ပါတယ်။ CSS data types တွေကို Mozilla Developer Network docs မှာ လေ့လာနိုင်ပါတယ်။ အဲ့မှာဆိုရင် CSS data types တွေကို list နဲ့ဖော်ပြပေးထားပါတယ်။ Data type တစ်ခုဟာ အမှန်တစ်ကယ်တော့ CSS value ရဲ့ type တစ်ခုလို့လဲပြောလို့ရပါတယ်။ အခြား data types တွေကတော့ integers, numbers and percentages တို့ပဲဖြစ်ပါတယ်။ သူတို့ဟာလည်း common data types တွေပဲဖြစ်ပါတယ်။ အဲ့မှာဆိုရင် positive or negative integer, number or percentage values တွေကို CSS properties အတွက်အသုံးပြုနိုင်ပါတယ်။ အများအားဖြင့်တော့ integer တု့ိ number တို့ဟာ length unit တစ်ခုနဲ့ တွဲ သုံးတာများပါတယ်။ Length unit ဆိုတာကတော့ length data type ရဲ့ part ပဲဖြစ်ပါတယ်။ သူ့ကိုတော့ pixel, ems or rems တို့နဲ့ number value တစ်ခုကို unit တစ်ခုရဲ့ measurement တစ်ခုကို specify လုပ်ဖို့ရန်အတွက်အသုံးပြုပါတယ်။ ဥပမာ ကျွန်တော်တို့ဟာ font-size: 90px ဆိုပြီးပေးလိုက်တယ်ဆိုပါစို့ အဲ့မှာဆိုရင် 90 ဆိုတာက number data type ဖြစ်ပြီးတော့ px ဆိုတဲ့ pixel ကတော့ length ပဲဖြစ်ပါတယ်။ အဲ့နှစ်ခုရဲ့ ကြားမှာ space ခံရေးလို့ မရပါဘူး။ နောက်ထပ် CSS data types တွေကတော့ images and gradients ပဲဖြစ်ပါတယ်။ သူတို့ကို element တစ်ခုရဲ့ background image or gradient ပေးချင်တဲ့အခါမှာ background-image property နဲ့ တွဲသံုံးရတာပဲဖြစ်ပါတယ်။ background image ကို defined လုပ်တဲ့နေရာမှာ ကျွန်တော်တို့ display လုပ်ချင်တဲ့ image ရဲ့ path or URL ကို ရေးပေးရမှာဖြစ်ပါတယ်။ font-size property မှာဆိုရင်လည်း ကျွန်တော်တို့ဟာ length or percentage data type ကိုအသုံးပြုနိုင်ပါတယ်။

Relative and Absolute Length Units

Number value တစ်ခုကို define လုပ်တဲ့အချိန်မှာ ကျွန်တော်ဟာ a length or unit of measurement အဲ့ value အတွက် specify လုပ်ပေးဖို့လိုပါတယ်။ CSS မှာဆိုရင် length ကို ဖော်ပြဖို့အတွက် မတူညီတဲ့ units တွေအများကြီးရှိပါတယ်။ Length units ဟာဆိုရင် အခြား units တွေထက် ပိုပြီးတော့ flexible ဖြစ်ပါတယ်။ ဥပမာ relative units and percentages units ဟာဆိုရင် အခြား length units တွေနဲ့ ဆက်ဆက် နေတာပဲဖြစ်ပါတယ်။ Developers တွေအတွက်ကတော့ အဲ့ဒါဟာ screen size and resolution ပေါ်မှာ မူတည်ပြီးတော့ content ကို layout ချတဲ့နေရာမှာ ပိုပြီးတော့ လွယ်ကူစေပါတယ်။ Absolute units ဟာဆိုရင် တစ်ခုနဲ့ တစ်ခု relation လုပ်တဲ့နေရာမှာအမြဲတမ်း fixed ဖြစ်ပါတယ်။ Relative မှာတော့ မလုပ်နိုင်ပါဘူး။ ကျွန်တော်တို့ဟာ font-size, padding, margin, width တု့ိအတွက် length or percentage တို့ကို CSS properties မှာအသုံးပြုနိုင်တယ်ဆိုတာကိုလေ့လာခဲ့ပြီးဖြစ်ပါတယ်။ Length units မှာဆိုရင် မတူညီတဲ့ several unit types တွေကို အသုံးပြုနိုင်ပါတယ်။ အဲ့ဒါတာတွေဟာဆိုရင် length units နဲ့ length units တို့ရဲ့ relative ပဲဖြစ်ပါတယ်။ Absolute units အကြောင်းကို first အနေနဲ့ပြောပြချင်ပါတယ်။ CSS မှာဆိုရင် absolute unit 6 ခုရှိပါတယ်။ ဒါပေမဲ့ အသုံးများတာကတော့ pixel unit ပဲဖြစ်ပါတယ်။ Pixel size ကိုအသုံးပြုတဲ့အခါမှာ screen မှာဘဲဖြစ်ဖြစ် browser window မှာဖြစ်ဖြစ် pixel ဟာအတူတူပဲဖြစ်နေပါမယ်။ CSS pixel ဟာဆိုရင် abstract reference unit တစ်ခုပဲဖြစ်ပါတယ်။ Devise တစ်ခု or hardware pixels တစ်ခုပေါ်မှာ အခြေခံပြီးတော့ CSS pixel ဟာဆိုရင် အဲ့ device ရဲ့ pixel density ထက်ပိုပြီးတော့ သိသိသာသာ လုပ်ဆောင်နိုင်ပါတယ်။ Device တစ်ခုရဲ့ pixel density ဟာဆိုရင် higher ဖြစ်နေတယ်ဆိုရင် pixel values ကို rescales လုပ်ပေးရမှာဖြစ်ပါတယ်။

Percentage Units

Number values တွေကို percentage units တွေနဲ့ အတူ ပြောင်းလဲပြီးတော့ define လုပ်နိုင်ပါသေးတယ်။ ကျွန်တော်တို့ဟာ percentage unit ကို CSS မှာအသုံးပြုတဲ့အချိန်မှာ percentage value ဟာဆိုရင် element's length တစ်ခုရဲ့ parent ကို measured လုပ်ပေးပါတယ်။ Width, margin, and padding တွေမှာဆိုရင် percentages ကို အသုံးပြုပါတယ်။ သူတို့ကို elements တွေရဲ့ သူတို့ရဲ့ container နဲ့ relative ဖြစ်နေတဲ့ size တွေကို ပြောင်းလဲနိုင်အောင် ပြုလုပ်ဖို့ရန်အတွက် အသုံးပြုပါတယ်။ ကျွန်တော်တို့ဟာ heading ကို wiidth: 50%; ပေးလိုက်တဲ့အခါမှာ heading ဟာဆိုရင် browser window ရဲ့ တစ်ဝက် 50% ကိုယူလိုက်မှာဖြစ်ပါတယ်။ Browser ကို resize လုပ်လိုက်ရင်လည်း browser window's width ရဲ့ 50% ပဲယူမှာဖြစ်ပါတယ်။ ပြီးရင် margin-left: 10%; ပေးလိုက်တဲ့အခါမှာဆိုရင် heading ဟာ browser window ရဲ့ left ကနေ 10% ကွာဟ သွားမှာဖြစ်ပါတယ်။ အခုကျွန်တော်ရှင်းပြခဲ့တဲ့ percentage တွေကို ကျွန်တော် example ရဲ့ တစ်ခြား section မှာ အသုံးပြုမှာဖြစ်ပါတယ်။ အဲ့မှာဆိုရင် class name တွေဖြစ်တဲ့ primary-content နဲ့ secondary-content ကို width: 60%; ပေးပြီးတော့ example အနေနဲ့ လုပ်ပြသွားမှာဖြစ်ပါတယ်။ အဲ့မှာဆိုရင်လည်း browser window's width ရဲ့ 60% ကိုပဲယူမှာဖြစ်ပါတယ်။ Percentage ကို font-size အတွက်လည်းအသုံးပြုနိုင်ပါတယ်။ Text ရဲ့ size ဟာဆိုရင် သူ့ parent ရဲ့ font-size နဲ့ directly related ဖြစ်နေပါတယ်။ ကျွန်တော်တို့ဟာ body ထဲမှာ font-size ကို 150% ပေးလိုက်တဲ့အခါမှာ HTML body ထဲမှာရှိတဲ့ text တွေအားလုံးဟာ body ရဲ့ default size ဖြစ်တဲ့ 16 pixels ထက် 1.5 times ကြီးသွားမှာဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ အခုလိုပဲ font-size ကို ကိုယ်ပြောင်းချင်တဲ့ paragraph တွေမှာလည်း ပြောင်းနိုင်ပါတယ်။

နောက်ထပ်ကတော့ text-decoration property ပဲဖြစ်ပါတယ်။ သူ့ကိုအသုံးများတာကတော့ links တွေမှာဆိုရင် underlines တွေကို remove လုပ်တဲ့နေရာ" none" ဆိုတဲ့ values ပေးပြီး အသုံးများတာပဲဖြစ်ပါတယ်။ အခုကျွန်တော်တို့ page ရဲ့ links တွေမှာ အဲ့ property ကို အသုံးပြုမှာပဲဖြစ်ပါတယ်။ text-decoration property ကို ဘယ် elements မှာမဆို အသုံးပြုနိုင်ပါတယ်။ အခုကျွန်တော်တို့ဟာ heading text ရဲ့ အောက်မှာ underline လေးတားတာကို လုပ်ပြသွားမှာဖြစ်ပါတယ်။ အဲ့မှာ သုံးတဲ့ value ကတော့ underline ပဲဖြစ်ပါတယ်။ နောက်ထပ် font-weight property ပဲဖြစ်ပါတယ်။ သူ့ကိုတော့ text ရဲ့ weight style ကို ဘယ်လောက် thick or ဘယ်လောက် thin ဖြစ်တာကို set လုပ်ဖို့ရန်အတွက်အသုံးပြုပါတယ်။ သူ့မှာသုံးတဲ့ common values နှစ်ခုကတော့ normal and ်bold ပဲဖြစ်ပါတယ်။ Headings အားလုံးကတော့ default ကကို bold ဖြစ်ပြီးသားဖြစ်ပါတယ်။ normal values ကိုအသုံးပြုတယ်ဆိုရင်တော့ font-weight ဟာ normal ပဲဖြစ်နေပါမယ်။ အခုကျွန်တော်ပြောခဲ့တဲ့ font-weight property ရဲ့ keywords values နှစ်အပြင် နောက်ထပ် values နှစ်ကိုလည်းသုံးပြုနိုင်ပါတယ်။ အဲ့ဒါတွေကတော့ lighter and bolder ပဲဖြစ်ပါတယ်။ အဲ့ values နှစ်ခုဟာ text element တစ်ခုရဲ့ inherited font-weight ပေါ်မှာမူတည်ပါတယ်။ အသုံးများတာကတော့ normal and bold ကိုအသုံးများပါတယ်။ ကျွန်တော်တို့ဟာ font-weight property မှာဆိုရင် numeric တွေကိုလည်းအသုံးပြုနိုင်ပါတယ်။ အဲ့ numeric တွေဟာ 100 to 900 အထိပဲဖြစ်ပါတယ်။ 100 ဟာဆိုရင် thinnest weight ဖြစ်ပြီးတော့ 900 ဟာ heaviest weight အတွက်ဖြစ်ပါတယ်။ 400 or 500 ကတော့ medium normal weight အတွက်ပဲဖြစ်ပါတယ်။ နောက်ထပ် font and text properties အကြောင်းကို web typography မှာ ဆက်လေ့လာလို့ရပါတယ်။

Font-family, Font-size, Font-style

Text properties လိုပဲ font properties မှာဆိုရင်လည်း ကျွန်တော်တို့ appearance of text တွေကို font-family, font-size, font-style and more တွေကို အသုံးပြုပြီးတော့ change လုပ်လို့ရပါတယ်။ First အနေနဲ့ ကျွန်တော်တို့ရဲ့ text font အတွက် font-family property ကိုအသုံးပြုမှာဖြစ်ပါတယ်။ အဲ့မှာဆိုရင် specific font family and generic family ဆိုပြီး font-family names နှစ်ခုရှိပါတယ်။ Specific မှာဆိုရင် Verdana, Arial, or Helvetica တို့ရှိပါတယ်။ Generic family မှာဆိုရင် serif, sans serif, or monospace ဆိုပြီးရှိပါတယ်။ Browser ရဲ့ default ဟာ serif font ပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ရဲ့ page မှာ font-family property ကိုအသုံးပြုမှာဖြစ်ပါတယ်။ Value ကို san-serif လို့ပေးကြည့်ပါမယ်။ သူ့မှာဆိုရင် finishing or ornamentation မရှိပါဘူး။ ကျွန်တော်တို့ဟာ generic font family names တွေကို MND docs မှာလေ့လာနိုင်ပါတယ်။ ကျွန်တော်တို့ဟာ font တစ်ခုထဲကိုပဲ သုံးထားမယ်ဆိုရင် ကျွန်တော်တို့ အသုံးပြုလိုက်တဲ့ font ကို browser ကနေ display လုပ်ပေးကအရေးကြီးပါတယ်။ Users တွေဘက်ကကြည့်မယ်ဆိုရင်လည်း ကျွန်တော်တို့ အသုံးပြုလိုက်တဲ့ fonts ကို သူတို့ရဲ့ computer မှာ installed လုပ်ထားဖို့လိုပါလိမ့်မယ်။ Computers တွေမှာတော့ fonts တွေအားလုံးတော့ မရှိနိုင်ပါဘူး။ ဒါကြောင့် ကျွန်တော်တို့ဟာ web-safe fonts ခေါ်တဲ့ system အားလုံးမှာရှိနိုင်တဲ့ ဟာကိုအသုံးပြုမှာဖြစ်ပါတယ်။ တစ်ချို့ case တွေမှာဆိုရင်တော့ web-safe fonts ကို fall back ဆိုပြီးတော့ define လုပ်ကြပါတယ်။ အဲ့ဒါကို font stack လို့ခေါ်ပါတယ်။ CSS font stack မှာဆိုရင် list of fonts တွေကို ဦးစားပေးထားတာပဲဖြစ်ပါတယ် သူ့ဟာ browser ကနေ user system မှာ install လုပ်ထားတဲ့ fonts ကို အတွေ့မချင်း cycle လို fonts ကို find လုပ်ပေးမှာဖြစ်ပါတယ်။

Line-height Property

Line height ဆိုတာကတော့ text ရဲ့ lines ကြားထဲမှာရှိတဲ့ vertical spacing ကို define လုပ်ပေးတာပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ရဲ့ page မှာ text lines တွေအများကြီးနဲ့ long paragraphs မှာဆိုရင် users တွေအနေနဲ့ read လုပ်ဖို့ရန်အတွက် အခက်အခဲဖြစ်စေပါတယ်။ ပိုပြီးတော့ ကြည့်ကောင်းတဲ့ line spacing တွေဟာ ကျွန်တော်တို့ရဲ့ text ကို read လုပ်တဲ့အချိန်မှာ ပိုပြီးတော့လွယ်ကူစေပါတယ်။ အဲ့ဒါကြောင့် ကျွန်တော်တို့ဟာ light-height property ကိုအသုံးပြုမှာဖြစ်ပါတယ်။ အဲ့ property ကိုအသုံးပြုပြီးတော့ text ရဲ့ lines တွေကြားထဲက vertical gaps ကို increase or decrease လုပ်နိုင်ပါတယ်။ ကျွန်တော်တို့ရဲ့ page မှာ default line height value ပါရှိပြီးသားဖြစ်ပါတယ်။ အဲ့ property ကို body element ထဲမှာပဲ page ရဲ့ line height အားလုံးကို set လုပ်ဖို့ရန်အတွက်အသုံးများပါတယ်။ line-height property ဟာဆိုရင် မတူညီတဲ့ values တွေကို accept လုပ်နိုင်ပါတယ်။ သူ့ရဲ့ default value ကတော့ normal ပဲဖြစ်ပါတယ်။ သူဟာဆိုရင် unit-less number value, pixel or percentage value တွေကို လည်း accept လုပ်နိုင်ပါတယ်။ နောက်ထပ် relative units တွေဖြစ်တဲ့ em or rems units ကိုလည်း accept လုပ်နိုင်ပါသေးတယ်။ အကောင်းဆုံးကတော့ unit-less numbers ကို line heights အတွက်အသုံးပြုတာပိုကောင်းပါတယ်။

line height ဟာ element တစ်ခုရဲ့ font size နဲ့ သူ့ရဲ့ scales နဲ့ proportional ဖြစ်ပါတယ်။ သူဟာ length values ကိုအသုံးပြုတာထက် ပိုပြီးတော့ predictable ဖြစ်နိုင်ပါတယ်။ ကျွန်တော်တို့ဟာ line-height property ကို unit-less value 1.5 ပေးလိုက်တဲ့အခါမှာ browser ဟာဆိုရင် line height ကို determine လုပ်ဖို့ရန်အတွက် element တစ်ခုချင်းစီရဲ့ font size ကို 1.5 နဲ့ multiply လုပ်လိုက်တာပဲဖြစ်ပါတယ်။ နောက်ဆုံးအနေနဲ့ကတော့ CSS မှာဆိုရင် handy shorthand property တစ်ခုရှိပါတယ်။ font properties တစ်ခုထဲမှာ values တွေကို တစ်ခါတည်းထည့်ပြီးတော့အသုံးပြုလို့ရပါတယ်။ ဥပမာ ကျွန်တော်တို့ဟာ font-weight, font-size, line-height and font-family တွေကို တစ်ခုထဲမှာရေးချင်တယ်ဆိုရင် သုံးရမဲ့ property က font ဖြစ်ပြီးတော့ နောက်က ကျွန်တော်တို့အသုံးပြုချင်တဲ့ သက်ဆိုင်ရာ values တွေကို ထည့်ပေးရမှာဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ font shorthand property ကိုအသုံးပြုတဲ့အချိန်မှာ အရေးကြီးဆုံးအချက်ကတော့ သူ့မှာ font-size နဲ့ font-family ကို define ပေးရပါမယ်။ font-style or font-weight value ကိုအသုံးပြုချင်တယ်ဆိုရင်တော့ အဲ့ဒါတွေကို font-size value ရဲ့ ရှေ့မှာ define လုပ်ပေးရမှာဖြစ်ပါတယ်။
