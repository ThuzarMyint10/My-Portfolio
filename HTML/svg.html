SVG Theory

ကျွန်တော်တို့ဟာ computer screen နီးနီးကပ်ကပ်ကြည့်လိုက်တဲ့အခါမှာ ကျွန်တော်တို့ကြည့်လိုက်တဲ့ image ဟာ squares ဖြစ်နေတာကိုတွေ့ရမှာဖြစ်ပါတယ်။ အဲ့ဒါကိုတော့ pixels လို့ခေါ်ပါတယ်။ Graphics တွေဟာဆိုရင် pixels တွေနဲ့ composed လုပ်ထားပြီးတော့ အဲ့ဒါကို raster graphics လို့ခေါ်ပါတယ်။ အစောပိုင်းတွေကဆိုရင် phones တို့ tablets တို့ desktop တို့မှာ မတူညီတဲ့ physical dimensions တွေမရှိပါဘူး။ ဒါပေမဲ့ သူတို့မှာ မတူညီတဲ့ pixel densities တွေရှိပါတယ်။ User တွေအတွက်ဆိုရင်တော့ pixel များတဲ့ screen ဟာပိုပြီးတော့ ကြည့်တဲ့အခါမှာ images တွေကို ပြတ်ပြတ်သားသားမြင်နိုင်မှာဖြစ်ပါတယ်။ အဲ့ဒါဟာ web designer တွေအတွက်တော့ problem တစ်ခုပဲဖြစ်ပါတယ်။ မတူညီတဲ့ screen sizes တွေကို responsive web design နဲ့ handled လုပ်ထားပါတယ်။ ကျွန်တော်တို့ဟာ image ကို deliver လုပ်ဖို့ရန်အတွက် enough resolution မရှိခဲ့ဘူးဆိုရင် ကြည့်ရတာ ဝါးနေမှာဖြစ်ပါတယ်။ နောက်ထပ် ကျွန်တော်ဟာ resolution အများကြီးနဲ့ image ကို deliver လုပ်မယ်ဆိုရင် အဲ့ဒါဟာ bandwidth တွေကို waste ဖြစ်စေပြီး browser ကို load လုပ်တဲ့နေရာမှာ slower ဖြစ်နိုင်ပါတယ်။ Scalable Vector Graphic လို့ခေါ်တဲ့ SVG ဟာဆိုရင် great resolution ပဲဖြစ်ပါတယ်။ သူဟာဆိုရင် JPEG or PNG file တွေလို pixels တွေနဲ့ composed မလုပ်ပါဘူး။ သူဟာ vectors တွေနဲ့ composed လုပ်ထားတာပဲဖြစ်ပါတယ်။ SVG file တစ်ခုရဲ့ vector တစ်ခုမှာဆိုရင် x and y coordinate ပါတဲ့ point တစ်ခုပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ အဲ့လို points တွေအများကြီး လုပ်ပြီးတော့လည်း အဲ့ points တွေကို connect လုပ်နို်င်ပြီးတော့ ကျွန်တော်တို့ဟာ shapes and outlines တွေကို လုပ်ဆောင်နိုင်ပါတယ်။ ကျွန်တော်တို့ဟာ အဲ့ SVG file ကို zoom ဆွဲလိုက်တဲ့အခါမှာ သူဟာ ရှင်းရှင်းလင်းလင်းမြင်နိုင်ပါတယ်။ သူဟာ pixelated မဖြစ်ပါဘူး။ Vectorized imagery ဟာဆိုရင် rasterizes pixel grid ကို automatically protected လုပ်ထားတာပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ images တွေမှာဆိုရင် SVG file တွေကိုပဲအသုံးပြုသင့်ပါတယ်။ Digital cameras တွေဟာဆိုရင်တော့ photo ကို vector image အနေနဲ့ ယူတာမဟုတ်ပါဘူး။ SVGs ကို intricate illustrations အတွက်အသုံးလို့ရပါတယ်။

Creating SVG

SVG ဟာဆိုရင် HTML လို XML markup language တစ်ခုပဲဖြစ်ပါတယ်။ သူ့ထဲမှာဆိုရင် မတူညီတဲ့ elements တွေ attributes တွေ ပါဝင်ပါတယ်။ ကျွန်တော်တို့ဟာ SVG file ကို HTML file လိုပဲ text editor ကိုအသုံးပြုပြီးတော့ create လုပ်နိုင်ပါတယ်။လူအများစုကတော့ Adobe Illustrator လို program ကိုအသုံးပြုကြပြီးတော့ အဲ့ဒါတွေကို SVG format နဲ့ export လုပ်ကြပါတယ်။ SVG ရဲ့ elements နဲ့ attributes တွေကို Mozilla DeveloperNetwork documentation မှာလေ့လာနိုင်ပါတယ်။

SVGs as Images

SVGs ကို web page မှာ add လုပ်တဲ့နေရာမှာ မတူညီတဲ့ နည်းလမ်းနှစ်ခုရှိပါတယ်။ တစ်နည်းကတော့ SVG markup ကို HTML document ထဲကို direct add တာနဲ့ နောက်တစ်ခုကတော့ SVGs ကို web page ထဲကို add တာပဲဖြစ်ပါတယ်။ အခုကျွန်တော်ပြောပြပေးမှာကတော့ SVGs ကို web page ထဲကို add လုပ်တာပဲဖြစ်ပါတယ်။ သူ့ကို PNGS or JPEGs တို့လို့ပဲအသုံးပြုသွားမှာဖြစ်ပါတယ်။ အဲ့လို add တဲ့နေရာမှာ image element ထဲမှာထည့်ပြီးတော့လည်း အသုံးပြုနိုင်ပါတယ်။ ကျွန်တော်တို့ဟာ SVG file ကို add မလုပ်ခင်မှာ အရင်ဆုံးလုပ်ရမှာကတော့ SVG optimization ပဲဖြစ်ပါတယ်။ သူ့ကို SVGO or SVG optimizer လို့ခေါ်တဲ့ GitHub မှာရရှိနိုင်ပါတယ်။ အဲ့ဒါဟာ graphical user interface ပဲဖြစ်ပါတယ်။ သူဟာ Mac and Windows နှစ်ခုလုံးမှာ အလုပ်လုပ်နိုင်ပါတယ်။ အဲ့မှာဆိုရင် svgo-gui ကို install လုပ်ပေးရမှာဖြစ်ပါတယ်။ Install တင်ပြီးရင် ကျွန်တော်တို့ သုံးချင်တဲ့ file ကို အဲ့ထဲကို ဆွဲထည့်လိုက်ရမှာဖြစ်ပါတယ်။ အဲ့လိုလုပ်တာဟာ ကျွန်တော်တို့ မလိုအပ်တဲ့ additional metadata ကို remove လုပ်ပေးတာပဲဖြစ်ပါတယ်။ အဲ့ဒါဟာ file size ကိုလည်း pretty small ဖြစ်အောင်လည်း လုပ်ပေးတာပဲဖြစ်ပါတယ်။

SVG Optimization

ကျွန်တော်အခုပြောပြပေးမဲ့ အကြောင်းကတော့ SVG ကို အသုံးပြုပြီးတော့ ကျွန်တော်တို့ရဲ့ page ကို ဘယ်လို improve ဖြစ်အောင် လုပ်မယ်ဆိုတာကိုပြောပြပေးမှာဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ SVG file ကို optimizing,့ organizing, and structuring လုပ်ဖို့ရန်အတွက် useful tools တွေကို လေ့လာရမှာဖြစ်ပါတယ်။ ကျွန်တော်ဟာ Bolt ကဲ့သို့သော် bill tool ကိုအသုံးပြုပြီးတော့ SVG icon system ကို ဘယ်လို create လုပ်တယ်ဆိုတာကိုလည်းပြောပြပေးမှာဖြစ်ပါတယ်။ Optimization ဟာဆိုရင် SVG တွေကို web ပေါ်မှာ display လုပ်ဖို့ရနအတွက် အရေးကြီးတဲ့ step တစ်ခုပဲဖြစ်ပါတယ်။ File size တွေကို smallest possible ဖြစ်အောင် လုပ်တာဟာ users တွေအတွက် ကျွန်တော်တို့ရဲ့ sites ကို faster ဖြစ်အောင်လုပ်ဆောင်ပေးနိုင်ပါတယ်။ SVG file size တွေကို reduce လုပ်ဖို့ဆိုရင် အသုံးများတဲ့ SVG optimization tool လို့ခေါ်တဲ့ SVGO ကိုအသုံးပြုမှဖြစ်ပါတယ်။ အဲ့ဒါကို github.com မှာ download လုပ်နိုင်ပါတယ်။ SVGO or SVG ဟာဆိုရင် node based command line tool တစ်ခုပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ အဲ့ဒါကိုအသုံးပြုမယ်ဆိုရင် ကျွန်တော်တို့ရဲ့ computer မှာ nodejs and NPM ကို installed လုပ်ထားဖို့လိုပါတယ်။ SVG editors တွေဟာ SVG file တစ်ခုကို exporting လုပ်ပြီးတဲ့ အချိန်မှာ lots of extra information တွေကို leave လုပ်ထားခဲ့ပါတယ်။ အဲ့ information ကတော့ ကျွန်တော်တို့ဟာ SVG file တစ်ခုကို receive လုပ်လိုက်တာပဲဖြစ်ပါတယ်။ SVGO ဟာဆိုရင် valuable time saving tool တစ်ခုပဲဖြစ်ပါတယ်။ အဲ့ဒါဟာ SVG workflow လိုပဲ ကျွန်တော်တို့ကို integrate လုပ်တဲ့နေရာမှာ လွယ်ကူပါတယ်။ SVGO command line tool ကို install လုပ်တဲအခါမှာ ကျွန်တော်တို့ရဲ့ terminal ကနေ npm install -svgo ကို ရိုက်ထည့်ပြီးတော့ install လုပ်နိုင်ပါတယ်။ SVG ရဲ့ list of commands တွေကို github.com ရဲ့ "Hoe to use" section မှာလေ့လာနိုင်ပါတယ်။

Grouping and Referencing With SVG

SVG မှာဆိုရင် circles path and polygons တို့လို drawing လုပ်ဖို့ရန်အတွက် elements တွေမပါဝင်ပါဘူး။ သူဟာဆိုရင် ကျွန်တော်တို့ရဲ့ SVG fiels ရဲ့ structure ကို organizing လုပ်ဖို့ရန်အတွက် useful elements တွေပါဝင်ပါတယ်။ အခုကျွန်တော်တို့ဟာ <g>, <defs>, <symbol> and <use> တို့ကို ဘယ်လို utilize လုပ်တယ်ဆိုတာကိုလေ့လာကြမှာဖြစ်ပါတယ်။ g or group element ဟာဆိုရင် SVG file ထဲမှာ grouping container တစ်ခုကနေနဲ့ လုပ်ဆောင်ပေးပါတယ်။ Grouping element ဟာဆိုရင် two or more SVG elements တွေကို group တစ်ခုဖြစ်စေပါတယ်။ Vector tools တွေမှာဆိုရင် SVG မှာ g tags ကိုအသုံးပြုဖို့ရန်အတွက် grouped objects တွေကို export လုပ်ပြီးသားဖြစ်ပါတယ်။ Def elements, stores elements and defines element ကိုသုံးတာဟာ ကျွန်တော်တို့ဟာ နောက်တစ်ကြိမ်မှာပြန်သုံးချင်တဲ့ အခါမှာ အသုံးပြုပါတယ်။ နောက်ထပ်တစ်ခုကတော့ use tag ပဲဖြစ်ပါတယ်။ သူဟာဆိုရင် selfclosing tag ပါပါတယ်။ Use tag ဟာဆိုရင် SVG elements တွေကို duplicates ဖြစ်စေပြီး အခြားနေရာတွေမှာ သူ့တို့ကို display လုပ်နိုင်ပါတယ်။ ကျွန်တော်တို့ဟာ reuse a single element or a group of elements inside g or defs tags duplicate လုပ်နိုင်ပါတယ်။

Symbol element defs and g elements တို့နဲ similar ဖြစ်ပါတယ်။ A symbol or groups elements ဟာဆိုရင် ဘယ်နေရာမှာဘဲဖြစ်ဖြစ် reference လုပ်နိုင်တဲ့ reasonable template တွေကို define လုပ်တဲ့နေရာမှာအသုံးပြုနိုင်ပါတယ်။ Symbol element ကို graphics အတွက်အသုံးပြုနိုင်ပါတယ်။ အဲ့ graphics ကို same document မှာ multiple times အသုံးပြုထားတာပဲဖြစ်ပါတယ်။ Symbol element မှာဆိုရင် g and defs elements ပေါ်မှာ significant advantage တစ်ခုရှိပါတယ်။

SVG: Sprite Tools

ကျွန်တော်တို့ဟာ simple element တစ်ခုကို referencing လုပ်ပြီးတော့ multiple places SVG graphics တွေကို ဘယ်လို display လုပ်တယ်ဆိုတာကို လေ့လာခဲ့ပြီးဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ SVG file ထဲမှာ image tag တွေနဲ့ icon တွေကို rendering လုပ်တာဟာ ကျွန်တော်တို့ရဲ့ HTTP loads ကို save လုပ်ပေးတာပဲဖြစ်ပါတယ်။ ကျွန်တော်တို့ဟာ SVG file တစ်ခုထဲကနေ ကျွန်တော်တို့ရဲ့ icons တွေကို serving လုပ်တာဟာ ကျွန်တော်တို့ရဲ့ sites ကို run တဲ့ အခါ ပိုပြီးတော့ faster ဖြစ်အောင်လုပ်ပေးနိုင်ပါတယ်။ SVG sprites တွေကို generate လုပ်ပေးတဲ့ tool ကို svg-sprite လို့ခေါ်ပါတယ်။ svg-sprite ဟာဆိုရင် ကျွန်တော်တို့ရဲ့ SVG file ကို optimizes လုပ်ပြီးတော့ သူတို့ကို SVG sprite sheet ထဲကို places လုပ်ပေးပါတယ်။ Best part ကတော့ create လုပ်ထားတဲ့ svg-sprite ကို Gulp ထဲကို port လုပ်လိုက်တာပဲဖြစ်ပါတယ်။ svg-sprite အကောင်းကို github.com ရဲ့ gulp-svg-sprite section မှာ လေ့လာနိုင်ပါတယ်။ အဲ့ထဲမှာပါတဲ့ usage instruction တွေအတိုင်း လုပ်ဆောင်ရမှာဖြစ်ပါတယ်။

3D in Browser

ကျွန်တော်တို့ အခုလေ့လာမှာကတော့ browser ထဲမှာရှိတဲ့ 3D browser အကြောင်းပဲဖြစ်ပါတယ်။ Web browser တွေဟာ 2D graphics ကိုပဲ သိကြတာများပါတယ်။ အခုဆိုတော့ တစ်ကမ္ဘာလုံးမှာ flat design ကိုအသုံးပြုနေကြပြီဖြစ်ပါတယ်။ ကျွန်တော်တို့မှာလဲ CSS ထဲမှာသုံးတဲ့ box model ရှိနေပြီဖြစ်ပါတယ်။ Browser တွေဟာ things တွေကို 2D နဲ့ displaying လုပ်တဲ့အခါမှာ ကောင်းကောင်းမွန်မွန် display လုပ်နိုင်ပါတယ်။ တစ်ချို့နေရာတွေမှာဆိုရင် အဲ့ဒါဟာ expansive ဖြစ်နိုင်ပါတယ်။ ဘာလို့လဲဆိုရင် အဲ့ဒါတွေအားလုံးဟာ Central Processing Unit လို့ခေါ်တဲ့ CPU ပေါ်မှာပဲ processing ဖြစ်တဲ့အတွက်ကြောင့်ပဲဖြစ်ပါတယ်။ Computer တွေမှာ ဆိုရင် CPU နဲ့ Graphic Processing Unit လို့ခေါ်တဲ့ GPU ပါကြပါတယ်။ Web browsers တွေမှာဆိုရင် GPU ရဲ့ advantage မရှိကြပါဘူး။ အဲ့ web browsers တွေဟာဆိုရင် CPU ကိုပဲအသုံးပြုကြတာများပါတယ်။ ကျွန်တော်တို့မှာ GPU ကို အသုံးပြုနိုင်တဲ့ ability ရှိတာဟာ ကောင်းမွန်ပါတယ်။ အဲ့ထဲမှာ ပါတဲ့ new technologies တွေဖြစ်တဲ့ three.js and webGL တို့ရဲ့ အသုံးပြုပုံကိုလည်းပြောပြပေးမှာဖြစ်ပါတယ်။ GPU ဟာဆိုရင် web browser ကို ဖြတ်ပြီးတော့ access မလုပ်နိုင်ပါဘူး ဒါပေမဲ့ ကျွန်တော်တို့ဟာ အခု WebGL ဆိုတဲ့ technology ကိုအသုံးပြုပြီးောတာ့ လုပ်နိုင်နေပါပြီ။ WebGL ဟာဆိုရင် သူ့ရဲ့ raw form ထဲမှာ အသုံးပြုဖို့အတွက် complex ဖြစ်ပါတယ်။ အဲ့ဒါဟာ OpenGL ရဲ့ port တစ်ခုပဲဖြစ်ပါတယ်။ OpenGL ဆိုတာ graphics technology တစ်ခုဖြစ်ပြီးတော့ သူ့ကို real-time 3D games တွေကို render လုပ်တဲ့ နေရာမှာ အသုံးပြုပါတယ်။ WebGL ရဲ့ matrix of support ကို 10UIs.com မှာ check out လုပ်နိုင်ပါတယ်။ Three.js ဆိုတာကတော့ framework တစ်ခုပဲဖြစ်ပါတယ်။ သူ့ကို three.js.org မှာလေ့လာနိင်ပါတယ်။
