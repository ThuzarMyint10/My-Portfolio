Introduction to CRUD

Dynamic application တစ်ခုရဲ့ heart ဟာဆိုရင် database တစ်ခုပဲဖြစ်ပါတယ်။ Example an e-commerce, sports team, or social network site or a productivity app on our phone တို့မှာဆိုရင် data တွေကို over time မှာ change လုပ်ဖို့လိုအပ်ပါတယ်။ Database တစ်ခုထဲမှာရှိတဲ့ data တွေပေါ်မှာ perform လုပ်ဆောင်ဖို့ရန်အတွက် operation 4 ခုရှိပါတယ်။ First, ကျွန်တော်တု့ိဟာ data တွေကို database တစ်ခုဆီကို create or add လုပ်နိုင်တာဖြစ်ပါတယ်။ Second ကတော့ data တွေကို read လုပ်ချင်တာဖြစ်ပါတယ်။ Reading မှာဆိုရင် searching and filtering result set တွေပါဝင်ပါတယ်။ Third ကတော့ any of the tables ထဲမှာ rows တွေကို update or edit လုပ်နိုင်တာဖြစ်ပါတယ်။ နောက်ဆုံးကတော့ data တွေကို remove or delete လုပ်နိုင်တာပဲဖြစ်ပါတယ်။ ဒီ operations 4 ခုကို CRUD နဲ့ shorthand ဖြင့် ခေါ်ကြပါတယ်။ ဒီ course မှာဆိုရင် database tables တွေကနေပြီးတော့ inserting, updating and delete rows of data ကို focus လုပ်သွားမှာဖြစ်ပါတယ်။ CRUD operations တစ်ခုချင်းစီမှာ သူတို့နဲ့ associated ဖြစ်တဲ့ SQL keyword တွေရှိပါတယ်။ ကျွန်တော်တို့ဟာ SELECT keyword ကိုအသုံးပြုပြီးတော့ database ကနေ data တွေကို reading လုပ်တာကိုလေ့လာခဲ့ပြီးဖြစ်ပါတယ်။

Adding a Row to a Table

CRUD operation တစ်ခုချင်းစီမှာဆိုရင် သူတို့ရဲ့ ကိုယ်ပိုင် SQL keyword တွေရှိကြပါတယ်။ ကျွန်တော်တို့ဟာ database ကနေ reading လုပ်အတွက် SELECT keyword ကို သိခဲ့ပြီးဖြစ်ပါတယ်။ အခု section မှာတော့ row တစ်ခုကို create လုပ်ဖို့ရန်အတွက် အသုံးပြုတဲ့ INSERT keyword အကြောင်းကို လေ့လာသွားမှာဖြစ်ပါတယ်။ အဲ့မှာဆိုရင် ကျွန်တော်တို့ စရမဲ့ keyword ကတော့ INSERT INTO table name ပဲဖြစ်ပါတယ်။ ဒါဟာဆိုရင် database ကို particular table တစ်ခုထဲကို some data တွေ inset လုပ်ချင်ပါတယ်ဆိုတာကို shows လုပ်တာဖြစ်ပါတယ်။ နောက် VALUES keyword ကိုလည်း ထည့်ပေးရမှာဖြစ်ပါတယ်။ သူ့ရဲ့နောက်မှာ ကျွန်တော်တို့ insert လုပ်ချင်တဲ့ values တွေကို comma နဲ့ separate လုပ်ပြီးတော့ ထည့်ပေးရမှာဖြစ်ပါတယ်။ နောက် အဲ့ values တွေကို parentheses နဲ့ surrounded လုပ်ပေးရမှာဖြစ်ပါတယ်။ ဒီ values တွေရဲ့ order ဟာ အရေးကြီးပါတယ်။ ကျွန်တော်တို့ဟာ same order schema ထဲမှာ values တွေကို inserted လုပ်မှာ columns တွေကို describes လုပ်ပေးမှာဖြစ်ပါတယ်။ နောက် ကျွန်တော်တို့ဟာ table name ရဲ့ နောက်မှာ parentheses နှင့် ကျွန်တော်တို့ insert လုပ်ချင်တဲ့ columns တွေကို comma နှင့် separate လုပ်ပြီးတော့ add လုပ်နိုင်ပါတယ်။ နောက် အဲ့ table name ရဲ့နောက်မှာ ထည့်မပေးလိုက်တဲ့ column မှာဆိုရင်တော့ values ဟာ null အဖြစ်နှင့် insert လုပ်သွားမှာဖြစ်ပါတယ်။

Adding Multiple Rows to a Table

နောက် ကျွန်တော်တို့ဟာ တစ်ကြိမ်ထဲမှာ multiple values တွေကို insert လုပ်ချင်တဲ့အခါမှာ query တွေကို over and over again ရေးနေမဲ့အစား ကျွန်တော်တို့ insert လုပ်ချင်တဲ့ values တွေကို comma နဲ့ separate လုပ်ပြီးတော့ အသုံးပြုနိုင်ပါတယ်။ Example INSERT INTO table name (<column 1>, <column 2>, ....) VALUES (<value 1>, <value 2>, .....), (<value 1>, <value 2>, .....), (<value 1>, <value 2>, .....)

Updating All Rows in a Table

ကျွန်တော်တို့ဟာ database ထဲမှာ row တစ်ခုကို INSERT keyword ကိုအသုံးပြုပြီးတော့ ဘယ်လို create လုပ်ရတယ်ဆိုတာကိုလေ့လာ ခဲ့ပြီးဖြစ်ပါတယ်။ Information တွေကို reading လုပ်ဖို့ရန်အတွက် SELECT keyword ကိုအသုံးပြုခဲ့ပြီးဖြစ်ပါတယ်။ Update မှာဆိုရင်လည်း ရှေ့ကကျွန်တော်တို့လေ့လာခဲ့တာတွေနဲ့မတူဘဲ အသုံးပြုရမဲ့ keyword ကတော့ UPDATE keyword ပဲဖြစ်ပါတယ်။ UPDATE statement ကို write လုပ်တဲ့အခါမှာ ရှေ့ဆုံးက UPDATE keyword နဲ့ နောက် ကျွန်တော်တို့ရဲ့ table name ပြီးတော့ SET keyword နှင့် သူ့ရဲ့နောက်မှာ ကျွန်တော်တု့ိ update လုပ်ချင်တဲ့ column name နဲ့ နောက် equal sign ခံပြီးတော့ နောက် ကျွန်တော်တို့ update လုပ်ချင်တဲ့ value ကို ထည့်ပေးရမှာဖြစ်ပါတယ်။ Example UPDATE <table> SET <column> = <value> ပဲဖြစ်ပါတယ်။ နောက် ကျွန်တော်တို့ဟာ တစ်ခုထက်ပိုတဲ့ columns တွေကို update လုပ်ချင်တဲ့အခါမှာ comma နဲ့ separate လုပ်ပေးရမှာဖြစ်ပါတယ်။ Example UPDATE <table> SET <column 1> = <value 1>, <column 2> = <value 2> ပဲဖြစ်ပါတယ်။

Updating Specific Rows

အခု ကျွန်တော်တို့ဟာ row တစ်ခုမှာရှိတဲ့ data တွေကို change လုပ်ဖို့ရန်အတွက်လေ့လာခဲ့ပြီးဖြစ်ပါတယ်။ အခု section မှာတော့ specific rows ကို target လုပ်ပြီးတော့ data တွေကို update လုပ်တာကို လေ့လာသွားမှာဖြစ်ပါတယ်။ ဒါဆိုရင် ပိုပြီးတော့ useful ဖြစ်ပါတယ်။ ဒါဟာဆိုရင် SELECT statement မှာ WHERE clause ကိုအသုံးပြုခဲ့သလိုပဲ UPDATE statements WHERE conditions ကို အသုံးပြုနိုင်ပါတယ်။ WHERE ရဲ့ conditions ဟာဆိုရင် shapes and sizes အားလုံးနဲ့ လာနိုင်ပါတယ်။ အဲ့မှာဆိုရင် operators တွေပါနိုင်ပါတယ်။ နောက် sets of values ထဲမှာ searching လုပ်တဲ့ IN keyword နဲ့လည်းလာနိုငပါတယ်။ နောက် BETWEEN keyword and AND keyword တို့လည်း အသုံးပြုနိုင်ပါတယ်။ နောက်ထပ်ကတော့ LINK keyword ကိုလည်းအသုံးပြုနိုင်ပါတယ်။

Removing Data from All Rows in a Table

Four CRUD operations ဆိုတာကတော့ create, read, update, and delete တို့ပဲဖြစ်ပါတယ်။ ကျွန်တော်တု့ိ first three keywords အကြောင်းကို လေ့လာခဲ့ကြပြီးဖြစ်ပါတယ်။ Update လိုပဲ delete's keyword ဟာဆိုရင် action ဟာ အတူတူပဲဖြစ်ပါတယ်။ Delete statement ကို DELETE FROM <table> ပဲဖြစ်ပါတယ်။ Delete keyword ဟာဆိုရင် table တစ်ခုကနေပြီးတော့ rows တွေကို removes လုပ်ပေးတာဖြစ်ပါတယ်။ သူဟာ empty ဖြစ်တဲ့ row's values တွေကိုလည်း removes လုပ်ပေးမှာဖြစ်ပါတယ်။ အဲ့မှာ ကျွန်တော်တို့သိထားရမှာကတော့ ကျွန်တော်တု့ိဟာ undo လုပ်လို့ရမှာမဟုတ်ပါဘူး။ နောက် ကျွန်တော်တို့ရဲ့ data တွေကို backup လုပ်မထားဘူးဆိုရင် ကျွန်တော်တို့ဟာ အဲ့ data တွေကို restore လုပ်နိုင်မှာမဟုတ်ပါဘူး။

Removing Specific Rows

အခု section မှာတော့ specific rows ကို target လုပ်ပြီးတော့ delete လုပ်တာကို လေ့လာသွားမှာဖြစ်ပါတယ်။ ဒါဆိုရင် ပိုပြီးတော့ useful ဖြစ်ပါတယ်။ ဒါဟာဆိုရင် SELECT and UPDATE statements မှာ WHERE clause ကိုအသုံးပြုခဲ့သလိုပဲ DELETE statements WHERE conditions ကိုအသုံးပြုနိုင်ပါတယ်။ WHERE ရဲ့ conditions ဟာဆိုရင် shapes and sizes အားလုံးနဲ့ လာနိုင်ပါတယ်။ အဲ့မှာဆိုရင် operators တွေပါနိုင်ပါတယ်။ နောက် sets of values ထဲမှာ searching လုပ်တဲ့ IN keyword နဲ့လည်းလာနိုငပါတယ်။ နောက် BETWEEN keyword and AND keyword တို့လည်း အသုံးပြုနိုင်ပါတယ်။ နောက်ထပ်ကတော့ LINK keyword ကိုလည်းအသုံးပြုနိုင်ပါတယ်။

Introduction to Transactions

အခု section မှာတော့ commitment အကြောင်းကို ပြောကြည့်ရအောင်။ ကျွန်တော်တို့ဟာ database တစ်ခုနဲ့ relationship တစ်ခုထဲမှာဆိုရင် ကျွန်တော်တို့ဟာ committed လုပ်ဖို့လိုပါတယ်။ Especially အရတော့ ကျွန်တော်တို့ဟာ data တွေကို modifying လုပ်တဲ့အချန်မှာပဲဖြစ်ပါတယ်။ ကျွန်တော်တု့ိဟာ CRUD operations ရဲ့ တစ်ခုကို performs လုပ်တဲ့ statement တစ်ခုကို write လုပ်တဲ့အချိန်မှာ အဲ့ SQL statement ဟာ certain ဖြစ်ဖို့လိုပါတယ်။ Production environment တစ်ခုမှာဆိုရင် ဒါဟာ earlier state ကို revert လုပ်ဖို့ရန်အတွက် difficult ဖြစ်နိုင်ပါတယ်။ ဒါဟာဘာကြောင့်လဲဆိုတော့ most circumstances ရှိတဲ့ ကျွန်တော်တို့ရဲ့ statements ဟာ autocommit mode မှာ run လုပ်တဲ့အတွက်ကြောင့်ပဲဖြစ်ပါတယ်။ Most database တွေမှာဆိုရင် default အရ autocommit switched ဟာ ရှိနေပြီးသားဖြစ်ပါတယ်။ ကျွန်တော်တို့ အခု section မှာလုပ်ဆောင်ခဲ့တဲ့ operations တွေဟာဆိုရင် autocommit တွေဖြစ်ပါတယ်။ ဆိုလိုတာကတော့ ကျွန်တော်တို့ statements ရဲ့ results တွေကို disk ထဲမှာ saved လုပ်ပေးတဲ့အတွက်ကြောင့်ပဲဖြစ်ပါတယ်။ Real world မှာဆိုရင် database seed scripts ဟာ hundreds and more of statements တွေပါဝင်နိုင်ပါတယ်။ Crashes and other interruption တွေမှာတွေ့ရနိုင်တဲ့ issues တွေနဲ့ deal လုပ်ဖို့ရန်အတွက် trying လုပ်တာဟာ pain တစ်ခုပဲဖြစ်ပါတယ်။ အဲ့ဒါကြောင့် aucommit ကို disable လုပ်နိုင်တဲ့ way တစ်ခုရှိပါတယ်။ အဲ့ဒါကတော့ transactions ကို အသုံးပြုတာပဲဖြစ်ပါတယ်။ အဲ့မှာဆိုရင် နားလည်ဖို့ရန်အတွက် important concept တစ်ခုကတော့ ကျွန်တော်တို့ရဲ့ statements ရဲ့ အစမှာ BEGIN TRANSACTION; ဆိုပြီးတော့ စပေးရမှာဖြစ်ပါတယ်။ ဒါဟာဆိုရင် database ကို ကျွန်တော်တို့ရဲ့ statements တွေအားလုံးဟာ တစ်ခုထဲမှာ အားလုံးကို commited လုပ်သင့်ပါတယ်ဆိုတာကို instructs လုပ်ပေးတာဖြစ်ပါတယ်။ တစ်ခါတစ်လေမှာတော့ shorthand ဖြစ်တဲ့ BEGIN keyword ကိုလည်းအသုံးပြုနိုင်ပါတယ်။

Databases with Frameworks
