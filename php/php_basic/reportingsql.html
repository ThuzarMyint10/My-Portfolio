Retrieving Results in a Particular Order

ကျွန်တော်တု့ိ data တွေကို specific order တစ်ထဲမှာ retrieve လုပ်ချင်တဲ့အခါမှာ ORDER BY ကိုအသုံးပြုပေးရမှာဖြစ်ပါတယ်။ သူ့ရဲ့နောက်မှာ ကျွန်တော်တို့ specific order မှာ retrieve လုပ်ချင်တဲ့ column name ကို ထည့်ပေးရမှာဖြစ်ပါတယ်။ အဲ့ column ရဲ့နောက်မှာ ကျွန်တော်တို့ဟာ specific order တစ်ခုကို ascending or descending တစ်ခုတစ်ခုမှာ retrieve လုပ်ချင်တဲ့အခါမှာ [ASC|DESC] ကိုထည့်ပေးရမှာဖြစ်ပါတယ်။ နောက်ကျွန်တော်တို့ဟာ multiple columns တွေကို specific order မှာ retrieve လုပ်ချင်တဲ့မှာ column တစ်ခုချင်းစီကို comma နဲ့ separate လုပ်ပေးရမှာဖြစ်ပါတယ်။ Example " SELECT <columns> FROM <table> ORDER BY <column> [ASC|DESC];" and " SELECT <columns> FROM <table> ORDER BY <column> [ASC|DESC] , <column 2> [ASC|DESC], <column ...> [ASC|DESC];" ။

Limiting the Number of Results

Databases ဟာ massive amount of data တွေကို store လုပ်ထားတာဖြစ်ပါတယ်။ တစ်ခါတစ်ရံမှာ ကျွန်တော်တို့ဟာ results အားလုံးကို bring မလုပ်ချင်တဲ့အခါမှာတွေရှိပါတယ်။ အဲ့အတွက် SQL ဟာ returned ပြန်လာတဲ့ amount of results ကို limit လုပ်ပေးနိုင်တဲ့ way တစ်ခုကို provides လုပ်ပေးထားပါတယ်။ အဲ့ဒါကတော့ LIMIT keyword ကိုအသုံးပြုတာပဲဖြစ်ပါတယ်။ LIMIT keyword ရဲ့နောက်မှာ ကျွန်တော်တို့ retrieve လုပ်ချင်တဲ့ rows အရေအတွက်ကို ထည့်ပေးရမှာဖြစ်ပါတယ်။ Example "SELECT * FORM <table> ORDER BY <column> DESC LIMIT 3;" ။ ကျွန်တော်တို့ဟာ WHERE clauses နှင့်လည်း results တွေကို limit လုပ်နိုင်ပါတယ်။ နောက် အခု LIMIT keyword ကို အသုံးပြုပြီးတော့လည်း results တွေကို limit လုပ်နိုင်ပါတယ်။ WHERE clause ကိုအသုံးပြုတဲ့အခါမှာ ကျွန်တော်တို့ filter လုပ်ချင်တဲ့ conditions ကို တိတိကျကျသိရမှာဖြစ်ပါတယ်။

Paging Through Results

Results တွေရဲ့ top set နဲ့ limiting လုပ်တာဟာ fine thing တစ်ခုဖြစ်ပါတယ်။ Example results တွေထဲက 50 အထိ skipped လုပ်ပြီးတော့ သူ့ရဲ့နောက် 50 results ကို retrieve လုပ်ချင်တယ်ဆိုပါစို့ ။ ကျွန်တော်တု့ိဟာ skipped လုပ်တဲ့ 50 ရဲ့ရှေ့မှာ OFF-SET ဆိုတဲ့ keyword ကိုအသုံးပြုပေးရမှာဖြစ်ပါတယ်။ Example "SELECT * FROM <table> LIMIT 50 OFF-SET 50" ဆိုပြီးတော့ create လုပ်ပေးရမှာဖြစ်ပါတယ်။

Adding Text Columns Together

ကျွန်တော်တု့ိ ပြီးခဲ့တဲ့ section မှာ column names တွေကို human readable ဖြစ်အောင် AS keyword ကို အသုံးပြုခဲ့တာဖြစ်ပါတယ်။ ကျွန်တော်တု့ိဟာ column or values တွေကို အတူတူဖြစ်အောင် operator တစ်ခုကို အသုံးပြုပြီးတော့ join လုပ်နိုင်ပါတယ်။ အဲ့ operator ကို concatenation operator လို့ခေါ်ပါတယ်။ Concatenate လုပ်တယ်ဆိုတာ things တွေကို link or join လုပ်ပေးတာဖြစ်ပါတယ်။ Concatenate operator ကတော့ "||" ပဲဖြစ်ပါတယ်။ Example <value or column> || <value or column> ။

ကျွန်တော်တု့ိဟာ database ထဲမှာရှိတဲ့ user name ရဲ့ length ကိုလိုချင်တဲ့အခါမှာ "LENGTH()" ကိုအသုံးပြုပေးရမှာဖြစ်ပါတယ်။ အဲ့ LENGTH() ထဲမှာ ကျွန်တော်တို့ သိချင်တဲ့ column name ကို ထည့်ပေးရမှာဖြစ်ပါတယ်။ ဒီ function ကို ways အမျိုးနဲ့ အသုံးပြုနိုင်ပါတယ်။ Example "SELECT LENGTH(<column>) AS <alias> FROM <table>;" and "SELECT <columns> FROM <table> WHERE LENGTH (<column>) <operator> <value>;" ။

ကျွန်တော်တို့ case of string ကို change လုပ်ဖို့ရန်အတွက် number of reasons တွေရှိပါတယ်။ Example column တစ်ခုမှာရှိတဲ့ text တွေကို uppercase or lowercase ကို change လုပ်တာတို့ပဲဖြစ်ပါတယ်။ အသုံးပြုရတဲ့ functions တွေကတော့ uppercase ကို change လုပ်ချင်တဲ့အခါမှာ UPPER(<column name>) and lowercase ကို change လုပ်ချင်တဲ့အခါမှာ LOWER(<column name>) တု့ိပဲဖြစ်ပါတယ်။

Replacing Portions of Text

Another handy way တစ်ခုဖြစ်တဲ့ portions of text ကို other values တွေနဲ့ replace လုပ်လို့ရတာပဲဖြစ်ပါတယ်။ ဒါဟာဆိုရင် web page တစ်ခုဆီကို styling ကို adding လုပ်ဖို့ရန်အတွက် ကောင်းပါတယ်။ Example ကျွန်တော်တို့ဟာ search results မှာ search term ကို code တွေကို adding လုပ်ခြင်းဖြင့် web browser မှာ appears ဖြစ်လာအောင် highlight လုပ်နိုင်ပါတယ်။ အဲ့ဒါကတော့ REPLACE()function ပဲဖြစ်ပါတယ်။ Example REPLACE( <value or column>, <target>, <replacement>) ပဲဖြစ်ပါတယ်။ အဲ့မှာဆိုရင် <target> ဟာ ကျွန်တော်တု့ိ replace လုပ်ချင်တဲ့ value ပဲဖြစ်ပါတယ်။ နောက် <replacement> ကတော့ ကျွန်တော်တို့ replacement လုပ်မဲ့ text ပဲဖြစ်ပါတယ်။

Count Function and DISTINCT

အခု section မှာတော့ numbers တွေအကြောင်းကို လေ့လာသွားမှာဖြစ်ပါတယ်။ အဲ့ဒါကတော့ COUNT() function ပဲဖြစ်ပါတယ်။ သူ့ကို many different ways တွေနဲ့ အသုံးပြုကြပါတယ်။ သူ့ရဲ့ common usages ကတော့ table ထဲမှာရှိတဲ့ rows အားလုံးကို counting လုပ်တာပဲဖြစ်ပါတယ်။ Counting ဟာ query တစ်ခုထဲမှာရှိတဲ့ number of results ဖြစ်ပါတယ်။ နောက် counting ဟာ table တစ်ခုထဲမှာရှိတဲ့ unique or distinct entries တစ်ခုဖြစ်ပါတယ်။ နောက် ကျွန်တော်တု့ိဟာ aggregates or collections of things တွေကိုလည်း count လုပ်နိုင်ပါတယ်။ Count ဟာဆိုရင် rows ထဲမှာရှိတဲ့ value တွေကိုပဲ counts လုပ်ပေးတာဖြစ်ပါတယ်။ နောက်ကျွန်တော်တု့ိဟာ duplicates ဖြစ်နေတဲ့ rows တွေရှိတဲ့အခါမှာ row အရေးအတွက်ကို သိချင်တဲ့အခါမှာ DISTINCT ကိုအသုံးပြုပေးရမှာဖြစ်ပါတယ်။ Example "SELECT DISTINCT category FROM products; " ။

Counting Groups of Rows

အခု section မှာတော့ group of rows ကို count လုပ်ဖို့ရန်အတွက် Count function ကို အသုံးပြုသွားမှာဖြစ်ပါတယ်။ Example category တစ်ခုချင်းစီမှာ products ဘယ်လောက်ရှိတယ်ဆိုတာပဲဖြစ်ပါတယ်။ အဲ့ဒါအတွက် GROUP BY ကို အသုံးပြုပေးရမှာဖြစ်ပါတယ်။ ဒါဟာဆိုရင် DISTINCT keyword နဲ့ တူပါတယ်။ ဒါပေမဲ့ key different တစ်ခုရှိပါတယ်။ DISTINCT ဟာ duplicates တွေကို discards လုပ်ပေးတာဖြစ်ပါတယ်။ Group by ကတော့ rows တွေကို အတူတူ groups လုပ်ပေးတာဖြစ်ပါတယ်။ နောက် DISTINCT ကို column တစ်ခုထဲမှာရှိတဲ့ unique values ကို get လုပ်ဖို့ရန်အတွက် အသုံးပြုတာဖြစ်ပါတယ်။ သူ့ကို query တစ်ခုရဲ့ SELECT portion ထဲမှာ အသုံးပြုရတာဖြစ်ပါတယ်။ နောက် GROUP BY ကတော့ statement ရဲ့ နောက်ဆုံးမှာ appears ဖြစ်လာတာဖြစ်ပါတယ်။ သူဟာ collections of rows ကို column တစ်ခုထဲမှာရှိတဲ့ particular value နဲ့ grouped လုပ်ပေးတာဖြစ်ပါတယ်။ Example "SELECT DISTINCT ≪column> FROM ≪table>; " and "SELECT ≪column> FROM ≪table> GROUP BY ≪column>; " ။

ကျွန်တော်တို့ဟာ rows တွေကို counted လုပ်ခဲ့ပြီးဖြစ်ပါတယ်။ Example ကျွန်တော်တု့ိဟာ e-commerce site database တစ်ခုနှင့် အလုပ်လုပ်နေတယ်ဆိုပါစို့ ။ CEO ဟာ gift card တစ်ခုနဲ့ most loyal customer ကို reward လုပ်ချင်တဲ့အခါမှာ ကျွန်တော်တို့ဟာ ဘယ် customer ဟာ ကျွန်တော်တု့ိရဲ့ site မှာ most money ကို spent လုပ်ထားတယ်ဆိုတာကို find out လုပ်ပေးရမှာဖြစ်ပါတယ်။ အဲ့အတွက် ကျွန်တော်တို့ဟာ SUM function ကိုအသုံးပြုပေးရမှာဖြစ်ပါတယ်။ သူ့ကို GROUP BY keywords နဲ့ column တစ်ခုရဲ့ sum total ကို calculate လုပ်ဖို့ရန်အတွက် conjunction လုပ်ပြီးတော့ အသုံးပြုနိုင်ပါတယ်။ ကျွန်တော်တု့ိ SUM function ကို GROUP BY keywords နဲ့ အသုံးပြုတဲ့အခါမှာ differently grouped rows ကနေပြီးတော့ total values ကို generate လုပ်နိုင်ပါတယ်။ နောက် ကျွန်တော်တို့ဟာ most money spent ဖြစ်တဲ့ customer ကို လိုချင်တာဖြစ်တဲ့အတွက်ကြောင့် ORDER BY ကိုလည်းအသုံးပြုပေးရမှာဖြစ်ပါတယ်။ နောက်ကျွန်တော်တို့ သူ့တစ်ယောက်ထဲကိုလိုချင်တာဖြစ်တဲ့အတွက်ကြောင့်LIMIT function ကိုလည်းအသုံးပြုပေးရမှာဖြစ်ပါတယ်။ နောက် most loyal customer ရဲ့ information ပေါ်မှာအခြေခံပြီးတော့ CEO ဟာ $250 ထက်ပိုတဲ့ anyone ကို gift တစ်ခုကို ပေးချင်တဲ့အခါမှာ WHERE keywords ကို အသုံးပြုတဲ့အခါမှာ "misuse of aggregate: SUM()" ဆိုတဲ့ error ကိုရရှိမှာဖြစ်ပါတယ်။ ဒါဟာ ဘာကြောင့်လဲဆိုတော့ WHERE conditions ဟာ things တွေကို together grouping မလုပ်ခင်မှာresult set or rows ကို filtered လုပ်တာဖြစ်တဲ့အတွက်ကြောင့်ပဲဖြစ်ပါတယ်။ SUM ဟာဆိုရင် aggregated values တွေပေါ်မှာပဲ အလုပ်လုပ်တာဖြစ်ပါတယ်။ အဲ့ဒါကြောင့် ကျွန်တော်တု့ိဟာ HAVING keyword ကို အသုံးပြုပေးရမှာဖြစ်ပါတယ်။ သူ့ကို GROUP BY keywords ရဲ့နောက်မှာ HAVING keyword ကိုအသုံးပြုပေးရမှာဖြစ်ပါတယ်။

Example Code

        
            SELECT SUM(cost) AS total_spend, user_id FROM orders 
                GROUP BY user_id
                HAVING total_spend > 250
                ORDER BY total_spend;

ကျွန်တော်တု့ိဟာ column တစ်ခုလုံးရဲ့ average amount ကို လိုချင်တဲ့အခါမှာ AVG function ကိုအသုံးပြုပေးရမှာဖြစ်ပါတယ်။ သူ့ကို average ရဲ့ aggregate function လို့လည်းခေါ်ပါတယ်။ ကျွန်တော်တို့ဟာ records ရဲ့ subset တစ်ခုပေါ်မှာ average ကို find လုပ်ချင်တဲ့အခါမှာ WHERE clause ကိုအသုံးပြုနိုင်ပါတယ်။

Code Example

        
            SELECT AVG(<numeric colun>) FROM <table>;
            SELECT AVG (<numeric colun>) FROM <table> WHERE <condition>;
            SELECT AVG (<numeric colun>) FROM <table> ORDER BY <column>;

Getting Minimum and Maximum Values

Example ကျွန်တော်တို့ largest values and smallest values တွေကို လိုချင်တဲ့အခါမှာ MAX and MIN functions တို့ကိုအသုံးပြုနိုင်ပါတယ်။ ဒီ functions နှစ်ခုလုံးဟာ AVG function နှင့် အလုပ်လုပ်ပုံချင်းတူပါတယ်။

Code Example

        
             MAX(<column>), MIN (<column>)

ကျွန်တော်တု့ိဟာ ပြီးခဲ့တဲ့ section တွေမှာ operators တွေအကြောင်းကို လေ့လာခဲ့ပြီးဖြစ်ပါတယ်။ အခု section မှာတော့ numeric types ပေါ်မှာ mathematical transformations တွေကို လုပ်ဆောင်ပေးတဲ့ operators တွေအကြောင်းကိုလေ့လာသွားမှာဖြစ်ပါတယ်။ အဲ့ဒါတွေကတော့ addition operator (+) , subtraction operator(-), multiplication operator (*) and division operator (/) တို့ပဲဖြစ်ပါတယ်။ နောက်ကျွန်တော်တု့ိဟာ decimal ရဲ့နောက်မှာ number ပါဝင်တာကို control လုပ်ဖို့ရန်အတွက် ROUND function ကိုအသုံးပြုနိုင်ပါတယ်။ ဒီ ROUND function ဟာ arguments နှစ်ခုယူပါတယ်။ First ကတော့ ကျွန်တော်တု့ိ round လုပ်ချင်တဲ့ value နဲ့ second ကတော့ ကျွန်တော်တို့ value ကို round လုပ်ချင်တဲ့ decimal ရဲ့ number ပဲဖြစ်ပါတယ်။

Code Example

        
             ROUND(<value>, <decimal places>)
